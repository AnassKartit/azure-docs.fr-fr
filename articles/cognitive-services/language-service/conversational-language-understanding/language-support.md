# <a name="conversational-language-understanding-language-support"></a>Prise en charge des langues par Compréhension du langage courant 

Utilisez cet article pour découvrir quelles langues naturelles sont prises en charge par la fonctionnalité Compréhension du langage courant.

### <a name="supported-languages-for-conversation-projects"></a>Langues prises en charge pour les projets de conversation

Quand vous créez un projet de conversation dans Compréhension du langage courant, vous pouvez spécifier la langue principale de votre projet. La langue principale est utilisée comme langue par défaut du projet.

Les langues prises en charge pour les projets de conversation sont :

| **Langue** | **Code de la langue** |
| --- | --- |
| Portugais brésilien | `pt-br` |
| Chinois | `zh-cn` |
| Néerlandais | `nl-nl` |
| Anglais | `en-us` |
| Français | `fr-fr` |
| Allemand | `de-de` |
| Goudjrati | `gu-in` |
| Hindi | `hi-in` |
| Italien | `it-it` |
| Japonais | `ja-jp` |
| Coréen | `ko-kr` |
| Marathi | `mr-in` |
| Espagnol | `es-es` |
| Tamoul | `ta-in` |
| Télougou | `te-in` |
| Turc | `tr-tr` |

#### <a name="multilingual-conversation-projects"></a>Projets de conversation multilingues

Quand vous activez plusieurs langues dans un projet, vous pouvez y ajouter des données dans plusieurs langues. Vous pouvez aussi entraîner le projet dans une langue et faire des prédictions immédiatement dans d’autres langues. La qualité des prédictions peut varier d’une langue à l’autre, et certains groupes de langues fonctionnent mieux que d’autres en ce qui concerne les prédictions multilingues.


## <a name="supported-languages-for-orchestration-workflow-projects"></a>Langues prises en charge pour les projets de workflow d’orchestration

|Langage| Code langue |
|---|---|
| Portugais brésilien | `pt-br` |
| Anglais | `en-us` |
| Français | `fr-fr` |
| Allemand | `de-de` |
| Italien | `it-it` |
| Espagnol | `es-es` |

Les projets de workflow d’orchestration ne peuvent pas être utilisés dans plusieurs langues.
