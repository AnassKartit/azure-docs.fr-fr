# <a name="legacy-devices-on-the-azure-certified-device-catalog"></a>Appareils hérités sur le catalogue Azure Certified Device

Vous avez peut-être remarqué sur le catalogue Azure Certified Device que certains appareils n’ont pas le contour bleu ou l’étiquette « Azure Certified Device ». Ces appareils (dénommés « appareils hérités ») ont été précédemment certifiés dans le cadre du programme hérité.

## <a name="certified-for-azure-iot-program"></a>Programme Certifié pour Azure IoT

Avant le lancement du programme Azure Certified Device, les partenaires en matériel pouvaient auparavant certifier leurs produits dans le cadre du programme Certifié pour Azure IoT. Le programme de certification Azure Certified Device réoriente sa mission pour proposer des promesses aux clients plutôt que des fonctionnalités techniques d’appareil.

Les appareils qui ont été certifiés pour IoT Hub apparaissent sur le catalogue Azure Certified Device en tant qu’appareils hérités. Cette étiquette indique que les appareils ont été précédemment qualifiés via le programme désormais hors service, mais qu’ils n’ont pas été certifiés via le nouveau programme Azure Certified Device. Ces appareils sont clairement indiqués dans le catalogue par leur absence de contour bleu et sont accessibles via le filtre « Appareils certifiés IoT Hub (existants) ».
