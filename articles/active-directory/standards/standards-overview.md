# <a name="configure-azure-active-directory-to-meet-identity-standards"></a>Configurer Azure Active Directory pour respecter les normes d’identité

Dans le monde actuel des infrastructures interconnectées, la conformité aux normes et aux infrastructures gouvernementales et sectorielles est souvent obligatoire. 

Les infrastructures de conformité peuvent s’avérer extrêmement complexes. Microsoft s’engage auprès des gouvernements, des autorités de réglementation et des organismes de normalisation pour comprendre et répondre aux besoins de conformité de sa plateforme Azure. Vous pouvez tirer parti de plus de [90 certifications de conformité Azure](../../compliance/index.yml). Parmi ces offres, beaucoup concernent des pays et régions spécifiques. Azure propose également 35 offres de conformité spécifiques aux secteurs clés suivants : santé, secteur public, finance, enseignement, fabrication et médias. 

## <a name="azure-compliance-provides-a-head-start"></a>La conformité Azure offre une longueur d’avance

La conformité est une responsabilité partagée entre Microsoft, les fournisseurs de services cloud (les CSP) et les organisations. Vous pouvez vous appuyer sur les certifications de conformité Azure comme base pour votre conformité, puis configurer Azure Active Directory afin de respecter les normes d’identité.

Les CSP, les organismes publics et tous ceux qui collaborent avec eux doivent souvent respecter des normes strictes pour une ou plusieurs administrations. Ces normes sont notamment les suivantes :
* [FedRAMP (US Federal Risk and Authorization Management Program)](/azure/compliance/offerings/offering-fedramp)
* [NIST (National Institute of Standards and Technologies)](/azure/compliance/offerings/offering-nist-800-53)

Les CSP et les organisations des secteurs tels que la santé et la finance doivent également respecter des normes sectorielles, par exemple : 
* [HIPPA](/azure/compliance/offerings/offering-hipaa-us)
* [Sorbanes-Oxley (SOX)](/azure/compliance/offerings/offering-sox-us)

Pour plus d’informations sur les infrastructures de conformité, consultez [Offres de conformité Azure](/azure/compliance/offerings/).
