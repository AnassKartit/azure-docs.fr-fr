# <a name="configure-azure-active-directory-to-meet-identity-standards"></a>Configurer Azure Active Directory pour respecter les normes d’identité

Dans le monde actuel des infrastructures interconnectées, la conformité aux normes et aux infrastructures gouvernementales et sectorielles est souvent obligatoire. 

Les infrastructures de conformité peuvent s’avérer extrêmement complexes. Microsoft s’engage auprès des gouvernements, des autorités de réglementation et des organismes de normalisation pour comprendre et répondre aux besoins de conformité de sa plateforme Azure. Vous pouvez tirer parti de plus de [90 certifications de conformité Azure](https://docs.microsoft.com/azure/compliance). Nos offres de conformité incluent de nombreux pays et régions spécifiques. Azure propose également 35 offres de conformité spécifiques aux secteurs clés suivants : santé, secteur public, finance, enseignement, fabrication et médias. 

## <a name="azure-compliance-provides-a-head-start"></a>La conformité Azure offre une longueur d’avance

La conformité est une responsabilité partagée entre Microsoft, les fournisseurs de services cloud (CSP) et les organisations. Vous pouvez vous appuyer sur les certifications de conformité Azure comme base pour votre conformité, puis configurer Azure Active Directory pour respecter les normes d’identité.

Les fournisseurs de services cloud (CSP), les organismes gouvernementaux et tous ceux qui collaborent avec eux doivent souvent respecter des normes strictes pour un ou plusieurs gouvernements, notamment :
* [FedRAMP (US Federal Risk and Authorization Management Program)](https://docs.microsoft.com/azure/compliance/offerings/offering-fedramp)
* [NIST (National Institute of Standards and Technologies)](https://docs.microsoft.com/azure/compliance/offerings/offering-nist-800-53)

Les fournisseurs de services cloud et les organisations des secteurs tels que la santé et la finance doivent également respecter les normes des normes sectorielles telles que : 
* [HIPPA](https://docs.microsoft.com/azure/compliance/offerings/offering-hipaa-us)
* [Sorbanes-Oxley (Sox)](https://docs.microsoft.com/azure/compliance/offerings/offering-sox-us)

Pour plus d’informations sur les infrastructures de conformité, consultez [Offres de conformité Azure](https://docs.microsoft.com/azure/compliance/offerings/).
