> [!TIP]
> Dans cet article, le contenu s’applique au stockage de Table Azure d’origine. Toutefois, les mêmes concepts s’appliquent à l’API Table d’Azure Cosmos DB plus récente. L’API Table de Cosmos DB offre des performances plus élevées et une plus grande disponibilité, une distribution mondiale et des index secondaires automatiques. Elle est également disponible en mode [serverless](../articles/cosmos-db/serverless.md) basé sur la consommation. Il existe des [différences de fonctionnalités](../articles/cosmos-db/table/introduction.md
) entre l’API Table d’Azure Cosmos DB et celle du Stockage Table Azure. Pour plus d’informations, consultez [API Table Azure Cosmos DB](../articles/cosmos-db/table-introduction.md). Pour faciliter le développement, nous fournissons maintenant un [Kit SDK Tables Azure](https://devblogs.microsoft.com/azure-sdk/announcing-the-new-azure-data-tables-libraries/) unifié qui peut être utilisé pour cibler à la fois le stockage Table d’origine et l’API Table Cosmos DB.
