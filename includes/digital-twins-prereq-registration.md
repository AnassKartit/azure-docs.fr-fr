Pour authentifier toutes les ressources utilisées dans cet article, vous avez besoin de **configurer une inscription d’application [Azure Active Directory (Azure AD)](../articles/active-directory/fundamentals/active-directory-whatis.md)** . Suivez les instructions fournies dans [Procédure : Créez une inscription d’application](../articles/digital-twins/how-to-create-app-registration-portal.md) pour configurer cela. 

Une fois que vous disposez d’une inscription d’application, vous avez besoin de l’**ID d’application (client)** , de l’**ID de répertoire (locataire)** et de la **clé secrète client** ([disponible sur le portail Azure](../articles/digital-twins/how-to-create-app-registration-portal.md#collect-important-values)). Prenez note de ces valeurs pour les utiliser ultérieurement afin d’accorder l’accès aux API Azure Digital Twins.
